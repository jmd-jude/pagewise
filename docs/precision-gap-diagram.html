Architecture Overview

Two Approaches to Document Extraction — Why the Method Determines the Result

Approach A

Enterprise AI — Single-Pass Oracle

Input
Full transcript, 300 pages
~150,000 tokens, delivered at once
single prompt
AI Model — Single Pass
Attention distribution
Page 1 — strong Page 300 — fades
Model must hold entire document in working memory simultaneously. Attention dilutes across 300 pages; prominent early topics dominate.
Output
Coverage ~60%
Fluent. Organized. Missing middle-transcript testimony on care concerns, financial matters, medications, property issues — consistently and silently.

Structured Pipeline — AI as Step, Not Oracle

Contextualize
Each page is sent with adjacent pages as context — the model sees a window, not an island. Enough to understand continuity, not enough to dilute attention.

03
Extract per unit
For each page: one focused question. Does a new topic begin here? If so, describe it and record the page range. Model operates on bounded, verifiable input.
AI — focused task
04
Assemble in code
Outputs collected, sorted, deduplicated, and formatted with page-level citations by software. No AI in the assembly step — deterministic and auditable.
Software — no AI
Mapping to expert witness work
◎
Substantive Omission
Enterprise AI doesn't flag what it skips. The output looks complete. The missing testimony on medication changes or property transfers appears nowhere — until opposing counsel asks.
◎
Verifiable citations
Each entry in the pipeline output cites the exact page range in the original transcript. Every claim is traceable, every gap is visible, every statement is defensible.
◎
Hallucination floor
When the model sees one page at a time, it has nowhere to fabricate from. It either finds a topic or it doesn't. The bounded context eliminates the conditions under which confabulation occurs.
